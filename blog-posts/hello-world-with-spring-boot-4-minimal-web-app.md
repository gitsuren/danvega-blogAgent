Hello World with Spring Boot 4

A compact, step-by-step guide to build a minimal Spring Boot 4 web app that returns "Hello, World!".

Note: Availability and requirements vary by release. Use start.spring.io and the official Spring Boot release notes to confirm Boot 4 status, the exact minimum JDK (many Boot 4 builds target Java 21+), and any migration notes.

Prerequisites
- JDK 21 or newer recommended (verify the exact minimum in the Boot 4 release notes).
- Maven or Gradle (or the wrapper generated by Initializr).
- An editor or IDE.
- Internet access for start.spring.io and dependency resolution.

Create the project (Spring Initializr)
- Go to https://start.spring.io
- Choose:
  - Project: Maven or Gradle
  - Language: Java
  - Spring Boot: 4.x (if not listed, pick the latest available Boot 4 milestone/RC or the latest stable Boot and adjust later)
  - Group: com.example
  - Artifact: hello-springboot4
  - Dependencies: spring-boot-starter-web
- Generate and unzip.

Maven build file (replace 4.x.x with the official Boot 4 version)
Parent:
<parent>
  <groupId>org.springframework.boot</groupId>
  <artifactId>spring-boot-starter-parent</artifactId>
  <version>4.x.x</version>
</parent>

Dependency:
<dependencies>
  <dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-web</artifactId>
  </dependency>
</dependencies>

Plugin (required to produce an executable jar with dependencies):
<build>
  <plugins>
    <plugin>
      <groupId>org.springframework.boot</groupId>
      <artifactId>spring-boot-maven-plugin</artifactId>
    </plugin>
  </plugins>
</build>

Gradle users: apply the Spring Boot plugin and let Initializr generate the build file, or follow the plugin coordinates in the official docs.

Main application class (src/main/java/com/example/DemoApplication.java)
package com.example;
import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;

@SpringBootApplication
public class DemoApplication {
  public static void main(String[] args) {
    SpringApplication.run(DemoApplication.class, args);
  }
}

Controller and endpoint (src/main/java/com/example/HelloController.java)
package com.example;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RestController;

@RestController
public class HelloController {
  @GetMapping("/hello")
  public String hello() {
    return "Hello, World!";
  }
}

Build and run
Maven:
- Run without packaging: mvn spring-boot:run
- Build an executable jar: mvn clean package
  - Output: target/hello-springboot4-0.0.1-SNAPSHOT.jar (name will match your project)
  - Run: java -jar target/hello-springboot4-0.0.1-SNAPSHOT.jar

Gradle (wrapper):
- Run without packaging: ./gradlew bootRun (Windows: gradlew.bat bootRun)
- Build: ./gradlew bootJar, then run the jar in build/libs with java -jar

Test the endpoint
- After startup, the app listens on port 8080 by default.
- Test: curl -s http://localhost:8080/hello
- Expected response: Hello, World!

Packaging and distribution
- Spring Boot builds a self-contained executable jar with an embedded web server (when the Boot plugin is applied).
- Run anywhere you have a compatible Java runtime: java -jar your-app.jar

Troubleshooting
- Port in use: stop the other service or set server.port in src/main/resources/application.properties.
- Java version mismatch: ensure your java -version meets the Boot 4 minimum.
- Dependency resolution errors: for Maven, try -U; for Gradle, refresh/reload the project.
- Migrating from Boot 2/3: follow the official Boot 4 migration guide for breaking changes.

Next steps
- Add tests with @WebMvcTest or an integration test using @SpringBootTest.
- Explore other starters (data, security, actuator).
- Put the project on GitHub.
